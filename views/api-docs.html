Introduction & Authentication

Our API uses the widely adopted stubs/handler_api.php format. It is fully compatible with major SMS verification bot software.

Authentication: All endpoints require your unique api_key to be passed as a query parameter.

Keep your API key absolutely secret. Do not expose it on the frontend.

Get Balance

Retrieve your current account balance.

GET /stubs/handler_api.php?api_key=$api_key&action=getBalance

Parameters

FieldTypeDescription
actionStringMust be getBalance
api_keyStringYour API Key

Response

ACCESS_BALANCE:100.20

Request a Number

Purchases a virtual phone number for a specific service and country.

GET /stubs/handler_api.php?api_key=$api_key&action=getNumber&service=$service&country=$country

Parameters

FieldRequiredDescription
actionYesMust be getNumber
serviceYesService identifier (e.g. wa). If you cannot send an operator param, prefix it here (e.g. 6-wa).
countryYesNumeric country ID (e.g. 21 for USA).
operatorNoProvider routing filter (e.g. any or 6).
maxPriceNoMaximum allowed price you consent to. Recommended to prevent overcharging.

Responses

CodeDescription
ACCESS_NUMBER:ORD-5K8H:125550198Success! Returns Order ID and the Phone number (digits only).
NO_NUMBERSNo numbers available for requested parameters.
NO_BALANCEInsufficient wallet balance.

Get SMS Status

Poll this endpoint to check if the SMS code has arrived for an active order.

GET /stubs/handler_api.php?api_key=$api_key&action=getStatus&id=$id

Parameters

FieldTypeDescription
actionStringMust be getStatus
idStringOrder ID returned by getNumber.

Responsess

CodeDescription
STATUS_OK:883190SMS received! Contains the verification code.
STATUS_WAIT_CODENumber active, waiting for SMS delivery.
STATUS_CANCELOrder was cancelled or timed out.

Batch Status Check (V2)

Check multiple IDs at once: action=getStatusV2&id=123,124

{ "123": "STATUS_WAIT_CODE", "124": "STATUS_OK:5489" }

Change Request Status

Action an active order (Cancel for refund, or request secondary SMS).

GET /stubs/handler_api.php?api_key=$api_key&action=setStatus&status=$status&id=$id

Parameters

FieldDescription
status8: Cancel Order.    3: Request another SMS.
idOrder ID.

Responses

ACCESS_CANCELSuccessfully cancelled and remaining balance was refunded.
ACCESS_CANCEL_ALREADYThe order was already cancelled previously.
ACCESS_RETRY_GETWaiting for next SMS (if status=3).
